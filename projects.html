UPS Delivery System

This package delivery service is local to Salt Lake City, Utah. The project is able to deliver 40 packages in a mileage efficient way. This UPS system utilizes a greedy algorithm paired with a nearest neighbor approach. The command line interface allows the user to track the movements of an individual package (preferably for the customer's use), or all of the packages being delivered that day. When the packages have all been delivered, the user is able to see how many miles each truck traveled, and the total mileage for the day. The restraints on this project is that there are 3 trucks, but only 2 drivers. Certain packages need to be delivered on the same truck, and some packages arrive late to the delivery hub.

Dropout Risk Analyzer

This program uses machine learning to detect if the current student is at risk of failing. Using over 6,500 student records (correlating to the sliders in the video), the "Dropout Risk Analyzer" is able to determine a confidence level for the current student. After deciding if the student is "Low-risk" or "High-risk", the program is able to generate multiple normal distribution graphs to compare with other students. This program is aimed to help professors, counselors, and mentors understand their individual students more, and provide students with additional resources if they are deemed to be "At-risk". This web application is programmed in Python.

Search-and-Rescue Robot

The robot maintains an internal representation of the disaster environment by dynamically updating a map based on real-time sensor data and object detection events. This representation is stored using structured data containers such as a dictionary, allowing for efficient categorization, lookup, and visualization. When the robot encounters physical debris through its sensingNose sensor, it retrieves the object's alias ("Cylinder3" for example) and logs its world coordinates in a dictionary called self.debrisMap. This enables the robot to avoid redundant logging and provides responders with a map of obstructed paths. The robot has 3 main sensors:

  1. Nose sensor: This sensor detects when the robot runs into debris, allowing him to try and wiggle out of the way, or do a complete 180-degree turn. Identifies debris and marks coordinates on the map.
  2. Fire sensor: Detects if a fire is within range of the robot; marks coordinates for where the fire was detected
  3. Person sensor: Detects injured people in a disaster; marks the location of the person for first responders.
